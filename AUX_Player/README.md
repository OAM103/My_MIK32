# Аудиоплеер на DIP‑MIK32 (SPI TFT LCD 3.5" ILI9486)

## Коротко о проекте
   Музыкальный плеер с графическим интерфейсом для отладочной платы DIP‑MIK32 и TFT 3.5" (ILI9486). Управление — через сенсорный экран. Воспроизведение — через 12‑бит DAC + Timer32 + DMA. Интерфейс рисуется через SPI + DMA.

## Содержимое репозитория (основное)
- main.c — инициализация, основной цикл, обработка тача/кнопок.  
- system_config.c/.h — настройка SPI0 (LCD), SPI1 (тач), DMA, Timer32, DAC.  
- lcd.c/.h — отрисовка, PrintMassDMA_*, ClearMassDMA и т.д.  
- touchpad.c/.h — чтение/фильтрация координат тачпанели, преобразование в экранные координаты.  
- button.c/.h — менеджер кнопок, скроллинг, отрисовка списка.  
- tracks.c/.h — ADPCM‑декодер, заполнение DAC‑буферов, переключение треков.  
- icons.c/.h, music.c/.h — ресурсы и данные треков.
- mik32_hal_lcd5110_bitmaps.h - работа со шрифтами и текстом на кириллице и латинице

## Что выполняет проект
- Отрисовка GUI на TFT ILI9486 (через SPI + DMA).  
- Сенсорная панель: чтение X/Y и фильтрация (На том же экране).   
- Настройка громкости (горизонтальная шкала).  
- Пуск / пауза воспроизведения.  
- Список треков и выбор трека кликом по строке.
- Отображать текущий трек/позицию.   
- Построчная подача данных в DAC через DMA (два буфера) — непрерывное воспроизведение при нормальных условиях.

## Нерешённые проблемы
1. Кнопки переключения треков (вперёд / назад) не работают корректно. Нажатия на иконки left/right не вызывают смены трека.  
2. При скроллинге (перетаскивании списка) воспроизведение трека прекращается, в то время как при настройке громкости воспроизведение не прекращается. Возможные причины: конфликт использования DMA/CPU при одновременной построчной отрисовке экрана (много вызовов HAL_DMA_Start + HAL_DMA_Wait), приоритеты DMA каналов, блокирующие операции отрисовки, ожидания в цикле.
3. При скроллинге заново перерисовываются все кнопки (полная перерисовка области) - это медленно и визуально некрасиво.  

## Как собрать и загрузить
1. Убедитесь, что у вас настроено окружение для сборки (toolchain/IDE для DIP‑MIK32).  
2. Настройте пути к HAL библиотекам (mik32_hal).  
3. Скомпилируйте проект (пример: используйте VSCode и PlatformIO).  
4. Загрузите прошивку на плату через ваш программатор/bootloader.

## Ключевые места в коде (коротко)

```

SPI1_Init(); // Touch (SPI1)
DMA_Init();
DAC_Init();
Timer32_Init();
HAL_DAC_Init(&hdac1);
```

```

    uint8_t byte = track->data[music_pos / 2];
```

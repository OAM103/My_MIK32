# Аудиоплеер на DIP‑MIK32 (SPI TFT LCD 3.5" ILI9486)

## Коротко  
   Музыкальный плеер с графическим интерфейсом для отладочной платы DIP‑MIK32 и TFT 3.5" (ILI9486). Управление — через сенсорный экран. Воспроизведение — через 12‑бит DAC + Timer32 + DMA. Интерфейс рисуется через SPI + DMA.

## Содержимое репозитория (основное)
- main.c — инициализация, основной цикл, обработка тача/кнопок.  
- system_config.c/.h — настройка SPI0 (LCD), SPI1 (тач), DMA, Timer32, DAC.  
- lcd.c/.h — отрисовка, PrintMassDMA_*, ClearMassDMA и т.д.  
- touchpad.c/.h — чтение/фильтрация координат тачпанели, преобразование в экранные координаты.  
- button.c/.h — менеджер кнопок, скроллинг, отрисовка списка.  
- tracks.c/.h — ADPCM‑декодер, заполнение DAC‑буферов, переключение треков.  
- icons.c/.h, music.c/.h — ресурсы и данные треков.
- mik32_hal_lcd5110_bitmaps.h - работа со шрифтами и текстом на кириллице и латинице

## Что выполняет проект
- Отрисовка GUI на TFT ILI9486 (через SPI + DMA).  
- Сенсорная панель: чтение X/Y и фильтрация (На том же экране).   
- Настройка громкости (горизонтальная шкала).  
- Пуск / пауза воспроизведения.  
- Список треков и выбор трека кликом по строке.  
- Построчная подача данных в DAC через DMA (два буфера) — непрерывное воспроизведение при нормальных условиях.

## Нерешённые проблемы
1. Кнопки переключения треков (вперёд / назад) не работают корректно.  
   - Симптом: нажатия на иконки left/right не вызывают смены трека.  
   - Вероятная причина: ошибка в проверке попадания по координатам в Buttons_ProcessTouch (перепутаны X/Y). См. раздел «Исправления и советы».

2. При скроллинге (перетаскивании списка) воспроизведение трека прекращается. При настройке громкости воспроизведение не прекращается.  
   - Симптом: во время скролла звук прерывается/замирает. При взаимодействии с вертикальной полосой громкости звука это не вызывает.  
   - Возможные причины: конфликт использования DMA/CPU при одновременной построчной отрисовке экрана (много вызовов HAL_DMA_Start + HAL_DMA_Wait), приоритеты DMA каналов, блокирующие операции отрисовки, ожидания в цикле. См. раздел «Отладка и рекомендации».

3. При скроллинге заново перерисовываются все кнопки (полная перерисовка области) — это медленно и визуально некрасиво.  
   - Решение — оптимизация отрисовки: частичная отрисовка видимой области, аппаратный скролл контроллера, кэширование изображений кнопок, throttle обновления.

Как собрать и загрузить
1. Убедитесь, что у вас настроено окружение для сборки (toolchain/IDE для DIP‑MIK32).  
2. Настройте пути к HAL библиотекам (mik32_hal).  
3. Скомпилируйте проект (пример: используйте Makefile/IDE — конфигурация для DIP‑MIK32).  
4. Загрузите прошивку на плату через ваш программатор/bootloader.

Рекомендации по исправлению известных ошибок и по отладке
- Исправить проверку попадания в кнопку (вероятная причина неработающих кнопок): в Buttons_ProcessTouch заменить ошибочную проверку (перепутаны X/Y) на корректную:
`c
int visible_y = btn->y - scroll_offset + WINDOW_Y;
if (touch_x >= btn->x && touch_x <= (btn->x + btn->width) &&
    touch_y >= visible_y && touch_y <= (visible_y + btn->height)) {
    if (btn->on_press) btn->on_press();
    return 1;
}
- Диагностика прерываний/паузы звука:
  - Добавить логи (xprintf) перед/после тяжёлых отрисовок (`DrawButtons`, `PrintMassDMA_*`) и в местах старта/окончания DMA, старта/остановки Timer32. Сравнить временные метки и выяснить, где звук «впадает» в паузу.
  - Временно закомментировать вызовы отрисовки в цикле и проверить, продолжается ли воспроизведение при скролле (это покажет, отрисовка ли виновата).
- Приоритеты DMA:
  - Убедитесь, что DMA канал аудио (Timer32 → DAC → hdma_ch1) имеет более высокий приоритет, чем DMA для SPI LCD (hdma_ch0). В `DMA_CH0_Init` / `DMA_CH1_Init` выставьте приоритеты соответственно.
- Минимизировать блокировки:
  - Заменить блокирующие `HAL_DMA_Wait` на неблокирующие (использовать callbacks по завершению DMA) или уменьшить время ожидания.
- Оптимизация отрисовки при скролле:
  - Рисовать только изменившуюся область, вычислять минимальную прямоугольную область изменений.  
  - Ввести throttle: обновлять экран не на каждое изменение координаты, а, например, с частотой не чаще 25–60 FPS:
c
static uint32_t last_draw = 0;
if ((HAL_GetTick() - last_draw) > 30) { DrawButtons(); last_draw = HAL_GetTick(); }
`
  - Использовать аппаратный скролл дисплея (если контроллер поддерживает) через `LCD_SetScrollArea` / `LCD_Scroll`, тогда не придётся перерисовывать содержимое при вертикальном смещении.
  - Кешировать изображения кнопок/строк в буфере и отправлять из памяти, не вычисляя каждый пиксель заново.

Частые вопросы (FAQ)
- Q: Как временно переключать треки, если кнопки не работают?  
  A: Выбирать трек можно из списка треков (нажатие на строку списка). Иконки left/right пока не работают из‑за ошибки проверки координат.
- Q: Почему звук останавливается при скролле, но не при регулировке громкости?  
  A: Вероятно, отрисовка списка запускает блокирующие DMA‑операции/ожидания, мешающие своевременной подаче данных в DAC; регулировка громкости затрагивает только переменные и малую отрисовку полосы и потому не блокирует аудио.

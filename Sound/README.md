# 🎧✨ Touch Sound Visualizer (MIK32 + ILI9488)

![photo_2026-04-14_11-20-32](https://github.com/user-attachments/assets/a754c8d8-6671-4b42-9f55-3d7742ca612b)

Интерактивный embedded-проект на базе микроконтроллера **MIK32 (DIP)** и TFT дисплея **ILI9488 (3.5", 480×320, SPI)**.

##  Идея проекта

Основная цель — **связать звук с действием пользователя** и сделать это не только функционально, но и визуально красиво.

Каждое касание экрана:

* создаёт **светящийся круг (огонёк)**
* воспроизводит **звук магического удара**

Проект реализован в двух вариантах:

1.  **Производительный (основной)**
2.  **Анимированный (визуально красивее, но тяжелее)**

---

##  Демонстрация

 При каждом касании:

* появляется «энергетическая точка»
* запускается звук
* можно нажимать сколько угодно быстро — система выдерживает нагрузку

---

##  Архитектура проекта

Проект состоит из трёх ключевых подсистем:

### 1.  Звук (DAC + Timer + ADPCM)

####  Как работает звук:

* Используется **12-битный DAC**
* Таймер генерирует прерывания (sample rate)
* В каждом прерывании:

  ```c
  hdac1.Instance_dac->VALUE = current_buf[buf_pos];
  ```

####  Двойная буферизация (double buffering)

```c
uint16_t dac_buf0[DAC_BUF_SIZE];
uint16_t dac_buf1[DAC_BUF_SIZE];
```

* Пока один буфер проигрывается, второй заполняется
* Это даёт **непрерывный звук без прерываний**

####  Формат звука — ADPCM

Почему ADPCM:

* занимает мало памяти
* дешёвый по вычислениям
* идеально для микроконтроллера

Декодирование:

```c
int16_t adpcm_decode_nibble(uint8_t nibble)
```

Что происходит:

* каждый байт = 2 сэмпла
* используется:

  * `predictor` — текущее значение сигнала
  * `step_table` — шаг изменения
  * `index_table` — адаптация шага

#### Генерация аудио

```c
void fill_dac_buf(uint16_t *buf)
```

Процесс:

1. читается ADPCM
2. декодируется в PCM
3. масштабируется (volume)
4. переводится в диапазон DAC (0–4095)

---

### 2.  Тачпад (SPI)

Чтение координат:

```c
HAL_SPI_Exchange(...)
```

Особенности:

* координаты 12-битные
* используются команды:

  ```c
  #define READ_X 0xD0
  #define READ_Y 0x90
  ```

####  Фильтрация

```c
read_filtered_coordinate()
```

* усреднение нескольких измерений
* убирает шум

####  Обработка касания

```c
void process_touchpad()
```

Логика:

* фиксируется **новое касание**
* предотвращается повтор при удержании
* запускается звук:

  ```c
  play_sound = 1;
  music_pos = 0;
  ```

---

### 3.  Графика (DMA + SPI)

####  Быстрый вывод через DMA

Все рисование идёт через:

```c
HAL_DMA_Start(...)
```

Это позволяет:

* не нагружать CPU
* рисовать очень быстро

---

## Отрисовка «огоньков»

Каждый огонёк состоит из **двух частей**:

### 1.  Градиентный круг

```c
void gradientCircle(...)
```

#### Как работает:

* используется **расстояние до центра**:

  ```
  dist² = x² + y²
  ```

* без sqrt → быстрее

#### Зоны:

| Область                     | Поведение   |
| --------------------------- | ----------- |
| внутри `r_circle`           | чистый цвет |
| между `r_circle` и `r_grad` | градиент    |
| вне `r_grad`                | фон         |

#### Интерполяция:

```c
R = r1 + ((r2 - r1) * t >> 8);
```

где `t` — коэффициент 0…255

 градиент полностью **без float и без деления в цикле**

---

### 2.  Контур круга

```c
void drawCircle(...)
```

Алгоритм:

* классический **Bresenham**
* рисует через горизонтальные линии
* использует:

```c
ClearMassDMA_Fast(...)
```

##  Вариант 1 — Производительный

* рисуется один круг на касание
* звук проигрывается стабильно
* можно нажимать **с любой частотой**
* звуки могут **накладываться**

✔ лучший по производительности
✔ без лагов

---

##  Вариант 2 — Анимация

Добавлена структура:

```c
typedef struct {
    int x;
    int y;
    int active;
} MovingCircle;
```

###  Массив кругов

```c
MovingCircle circles[MAX_CIRCLES];
```

###  Логика:

1. при касании создаётся круг

2. каждый кадр:

   ```c
   update_circles();
   ```

3. круги двигаются вниз:

   ```c
   circles[i].y += 2;
   ```

4. удаляются за экраном

---

###  Ограничение

При большом количестве кругов:

* падает FPS
* появляются задержки звука

 причина:

* тяжёлый рендер (DMA + вычисления)
* нет аппаратного ускорения
